# GUI Build Plan

## Project Overview
GUI application using wxWidgets and OpenCV with C++20 modules system. The application provides image processing capabilities with a main interface, image viewer, and zoom functionality.

## 최종 앱의 형태
앱은 이미지를 로드하고, 로드된 이미지에서 특정 영역을 선택할 수 있는 기능을 제공합니다. (여러개의 바운딩박스)
선택된 영역은 bounding box로 표시되며, 선택된 영역을 기반으로 추가적인 확대용 윈도우창을 생성해서 더 정교한 학습용 데이터를 얻을 수 있도록 합니다. (기본 윈도우창에서도 동일하게 데이터를 얻을 수 있게만듬).
바운딩박스로 찾아진 영역은 학습용 데이터 또는 매칭알고리즘으로 다른이미지에서 찾는데 사용할 수 있도록 저장할 수 있습니다.
학습용 데이터를 얻게 된 이후에 여러가지 이미지프로세싱 (템플릿 매칭, 특징점 매칭, 이진화, 외곽선 검출) 과 AI적용 또한 가능해야함.

## 기술 스택
- **GUI**: wxWidgets
- **이미지 처리**: OpenCV
- **언어**: C++20 modules
- **패턴**: MVC, Repository, Dependency Injection, Observer


## 현재 상태 및 제약사항
1. 픽셀단위 세그멘테이션 기능추가를 위해 바운딩박스 표시 체크박스 를 오디오 버튼으로 바꾸고 그 옆에 픽셀세그멘테이션 오디오버튼을 추가합니다. 이 둘은 동시에 활성화 될 수 없습니다. (모두 비활성은 가능해야함.)
2. 1차 확대된 관심영역을 스크롤윈도우 이미지로 변경해서 진행하는 기능을 추가해야함. 기존의 모든 기능들은 모두 동일하게 사용할 수 있어야하고, 확대된 이미지영역에서 그려지는 바운딩 좌표들은 원본 이미지에서의 좌표로 저장되어야함.
3. 확대 전의 원본이미지로 다시 되돌아가는 버튼도 추가해야함. 
5. 바운딩박스 리스트에는 여전히 확대 전의 바운딩박스 좌표들이 있을 것인데, 1차 관심영역 밖의 바운딩박스는 리스트뷰에서 색으로 표시하여 현재 보여질 수 없다는 것을 알게 함.

동작을 제대로해야하는데..
0-2. 스크롤윈도우 처음 이미지 로딩부분 리팩토링하기. 단일화. 개발하면서 기능이많아지면서 제대로 분리가 안되었음. 최초구현시 기능들이 여러 함수에 흩어져있음. setImage, setImageDataByID, OnPaint 등 디버깅이어려움

2. 픽셀 세그멘테이션 모드를 추가하여 바운딩박스 로직을 비활성하고 마우스 좌클릭 시 픽셀 한 칸 한 칸마다 세그멘테이션을 할 수 있도록 기능을 추가합니다. 해당 기능은 물체의 외곽선을 빠르게 추출하는 데 도움이 될 예정입니다.

4. 세그멘테이션을 빠르게 도와줄 기능을 추가해야함. 빠른 GrabCut 알고리즘 구현. 외곽선 검출, 외곽선 매부 채우기 및 작은 노이즈 제거(모폴로지, 레이블링 등 응용)

## 최종 아키텍처
```
GUI → ScrolledImageWindow → ImageEditController → ImageDataRepository → ImageData (데이터만)
                                              ↓
                                      ImageProcessor (모든 처리/렌더링)
                                              ↓
                                      GUI에 최종 결과 전달
```

## 각 계층의 책임
- **GUI (ScrolledImageWindow)**: UI 로직 및 사용자 상호작용, Controller를 통해 화면에 그릴 이미지 데이터 얻기
- **Controller**: 비즈니스 로직 및 데이터 흐름 제어, ImageProcessor를 통한 처리 파이프라인 조율
- **Repository**: ImageData 저장/조회/삭제 관리
- **ImageData**: 순수 데이터 모델 (원본 이미지, 바운딩박스, 메타데이터)
- **ImageProcessor**: 모든 이미지 처리 (렌더링, 이진화, 외곽선 검출, 템플릿 매칭 등)
