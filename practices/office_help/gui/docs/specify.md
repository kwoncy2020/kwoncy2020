# 각 에이전트별 자신이 다루어야할 부분을 처리 전 반드시 참조해야함.

## 컨트롤러
1. Controller를 추가하여 GUI부분에서 관심사를 분리할 예정입니다.
2. 파일로드로 얻게된 이미지데이터에 대한 정보는 ImageData클래스가 관리합니다.
3. Controller는 Repository를 통해 ImageData를 관리합니다.
4. Controller는 GUI와 독립적으로 동작해야 합니다.

## 모든GUI 상태값의 변경 규칙 (옵저버패턴관련 규칙)
1. 모든 패널 캡슐화와 관심사 분리를위해 다른 패널과 직접적으로 접근하지 않고 Controller를 통해 접근해야 합니다.
2. 옵저버 패턴은 무한루프나 한번의 값의 변경이 여러번의 동일한 값을 변경 시킬 수 있는 문제가 있으므로 이를 방지하기위해 아래의 규칙을 반드시 지켜야 합니다.
3. 컨트롤러로부터 온 값으로 데이터를 업데이트 후 다시 컨트롤러가 알림을 발생시키는 함수를 호출하지 말것.
4. 사용자의 입력으로 변경이 발생한 경우에만 컨트롤러가 알림을 발생시키는 함수를 호출할 것.




## 스크롤윈도우 패널
1. 스크롤윈도우에 마우스 좌클릭 시 해당 위치의 픽셀을 세그멘테이션하는 기능을 추가해야 합니다.
1-1. 스크롤윈도우에 세그멘테이션이미지를 빠르게 보여주기위해 RGB형식으로 가지고 있어야하며 직접 사용자의 클릭한 픽셀의 값을 변경하고, 필요하면 ImageProcesor클래스의 도움을 받도록함. 이때, ImageProcessor에서 받는 파라미터는 이미지의 format이 있어야함.
2. 빠른 반응속도를 위해 세그멘테이션이 진행되고 있는 이미지를 내부에 멤버변수로 저장해야합니다. 이미지는 원본을 복사하여 별도로 저장해 유지합니다.
