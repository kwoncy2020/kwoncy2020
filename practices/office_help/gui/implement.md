# 코드 작성시 아래의 내용을 꼭 지켜주세요.
- Clean Code원칙을 준수하고 최신의 아키텍쳐와 확장가능성 간결성 등을 고려할 것.
- 관심사의 분리(Separation of Concerns)를 지키고, 각 클래스나 함수는 단일 책임 원칙(SRP)을 따를 것.
- 의존성 주입(Dependency Injection)을 사용하여 결합도를 낮출 것.
- Factory Pattern을 사용하여 새로운 기능을 추가할 때 수정이 최소화되도록 설계할 것.
- 옵저버 패턴(Observer Pattern)을 사용하여 상태 변경 시 자동으로 UI를 업데이트할 것.

- gui부분과 로직을 분리해야하고 로직은 image_processor.cpp에 구현합니다.
- 해당 gui앱은 딥러닝 학습용데이터생성을 위해 빠르게 작업할 수 있도록 이미지의 일부 범위를 지정해서 확대하여 레이블을 정확하게 지정할 수 있도록 해야합니다.
- 또한 opencv 패턴매칭 알고리즘을 사용하여 사전에 레이블된 특정 패턴을 이미지에서 빠르게 찾을 수 있도록 해야합니다.
